# Pass110 Rejection: Positive Short Integer Parse Fast Path

## Target

- Bead: `frankenredis-zhphm.7`
- Profile source: pass109 SET P16/C50 profile after the owned writer boundary.
- Hot rows:
  - `fr_store::canonical_string_value_from_slice`: 7.35% self
  - `Store::set_plain_borrowed`: 5.24% self
  - Read/parse rows were below the keep threshold after pass108.

## Candidate Lever

Add a short positive integer fast path to `parse_i64` for canonical decimal
inputs starting with `1..=9` and length `<=18`, avoiding the generic signed
overflow path for common SET values generated by `fr-bench --datasize 3`.

This was the only production lever tested in this pass.

## Benchmark Gate

Baseline-only SET P16/C50/1M was noisy:

- Baseline standalone: `1.886s +/- 0.371s`.

Paired SET P16/C50/1M:

- Baseline: `1.0938954790700002s +/- 0.06465636159534638s`
- Candidate: `1.0204414378200002s +/- 0.050826107748506794s`
- Candidate looked `1.07x +/- 0.08` faster.

Reversed SET P16/C50/1M:

- Candidate: `1.222463627945s +/- 0.17525205250515802s`
- Baseline: `1.09184196182s +/- 0.04005073036508224s`
- Baseline was `1.12x +/- 0.17` faster.

The order-reversed run does not support a real win, so the candidate fails the
confidence gate.

## Decision

Rejected. The source hunk and candidate-only test were removed before commit.

Score estimate: Impact `1.0` x Confidence `0.5` / Effort `1.0` = `0.5`, below
the `>=2.0` keep gate.

  request-scoped command/value packets.
- Concrete target: profile keyspace probe/cache-miss/allocation telemetry around
  `Store::set_plain_borrowed`, then test one structural map/value layout lever
  only if the profile confirms probe or allocation churn is still dominant.
